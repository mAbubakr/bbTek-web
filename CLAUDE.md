## Architecture

This is an **Astro 5** site (BBTek.ca) built from the Astroplate boilerplate, using **React** for interactive islands, **Tailwind CSS v4** for styling, and deployed to **Cloudflare Pages** via Netlify.

### Configuration layer (`src/config/`)

All site-wide settings live in JSON files ‚Äî no `.env` required for most config:

- `config.json` ‚Äî site URL, feature flags (`search`, `theme_switcher`, `sticky_header`, etc.), nav button, Google Tag Manager, Disqus, and metadata
- `theme.json` ‚Äî design tokens (colors, fonts, type scale). **Do not edit `src/styles/generated-theme.css` directly** ‚Äî it is auto-generated by `scripts/themeGenerator.js` from `theme.json` on every `dev`/`build`
- `menu.json` ‚Äî navigation structure
- `social.json` ‚Äî social links

### Content layer (`src/content/`)

Content is stored as Markdown/MDX and loaded via Astro's Content Collections API (`src/content.config.ts`). Collections: `blog`, `projects`, `authors`, `homepage`, `about`, `contact`, `pages`, `ctaSection`, `testimonialSection`.

- Index/section files use the `-index.md` naming convention and are excluded from individual page generation
- Blog posts support `categories`, `tags`, and `author` frontmatter
- The `homepage/-index.md` frontmatter drives the entire homepage layout (banner, features, social_proof, benefits, projects sections)
- Blog content is also compiled to `.json/posts.json` and `.json/search.json` by `scripts/jsonGenerator.js` for client-side search

### Layouts & components (`src/layouts/`)

- `layouts/components/` ‚Äî Astro components (cards, pagination, logo, etc.)
- `layouts/partials/` ‚Äî Astro page regions (Header, Footer, PageHeader, etc.)
- `layouts/helpers/` ‚Äî React TSX islands (SearchModal, Announcement, DynamicIcon, Disqus)
- `layouts/shortcodes/` ‚Äî React TSX components auto-imported into MDX files (Button, Accordion, Notice, Video, Youtube, Tabs/Tab)

### Path aliases

Defined in `tsconfig.json`:
- `@/components/*` ‚Üí `src/layouts/components/*`
- `@/shortcodes/*` ‚Üí `src/layouts/shortcodes/*`
- `@/helpers/*` ‚Üí `src/layouts/helpers/*`
- `@/partials/*` ‚Üí `src/layouts/partials/*`
- `@/*` ‚Üí `src/*`

### Image handling

The image service is set to `noop` (passthrough) in `astro.config.mjs` because Cloudflare Workers does not support Sharp. Avoid using Astro's `<Image />` optimization component ‚Äî use plain `<img>` tags or the custom `ImageMod.astro` component instead.

### Deployment

- **Cloudflare Pages** is the target runtime (`wrangler.jsonc` present)
- **Netlify** config (`netlify.toml`) is also present as an alternative; build command is `yarn build`, publish dir is `dist`
- Node version pinned to 22.20.0
